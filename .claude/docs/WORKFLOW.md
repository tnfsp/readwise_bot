# 每日資訊處理流程 (Workflow)

> 由 `/pm` 維護

---

## 設計原則

1. **彈性優先**：可以一次完成，也可以分段執行
2. **時間約束**：總計 30-60 分鐘，不超時
3. **輸出導向**：每天至少要有一個輸出
4. **允許略過**：忙碌日可以跳過，不累積壓力

---

## 標準流程（完整版，50 分鐘）

```
┌─────────────────────────────────────────────────────────────────┐
│  Phase A: 掃描（10-15 分鐘）                                     │
│  ─────────────────────────────────────────────────────────────  │
│  目標：快速瀏覽，決定今日焦點                                    │
│                                                                 │
│  1. 打開 Telegram 推播（或 Reader）                              │
│  2. 快速掃描標題與摘要                                          │
│  3. 標記 1-3 篇「今日必讀」                                      │
│  4. 其餘標記「略過」或「稍後」                                   │
│                                                                 │
│  ⚡ 決策標準：                                                   │
│  • 必讀：與當前關注議題相關 / 可能改變我的想法 / 可立即應用      │
│  • 稍後：有趣但不急 / 需要更多時間消化                          │
│  • 略過：已知內容 / 與目標無關 / 信號弱                         │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Phase B: 深讀（20-30 分鐘）                                     │
│  ─────────────────────────────────────────────────────────────  │
│  目標：專注閱讀，提取洞見                                        │
│                                                                 │
│  1. 打開 Readwise Reader                                        │
│  2. 閱讀標記的「今日必讀」（1-3 篇）                             │
│  3. 邊讀邊 highlight 關鍵段落                                    │
│  4. 用 Reader 筆記功能記錄初步想法                               │
│                                                                 │
│  📝 閱讀時問自己：                                               │
│  • 這篇的核心論點是什麼？                                        │
│  • 我同意嗎？為什麼？                                            │
│  • 這跟我之前知道的有什麼不同？                                  │
│  • 這能應用在什麼地方？                                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Phase C: 輸出（10-15 分鐘）                                     │
│  ─────────────────────────────────────────────────────────────  │
│  目標：將閱讀轉化為自己的知識                                    │
│                                                                 │
│  1. 打開 Heptabase                                              │
│  2. 為每篇深讀的文章建立卡片（至少 1 張）                        │
│  3. 標記信心度（低/中/高）                                       │
│  4. 連結到相關的舊卡片                                          │
│                                                                 │
│  🎯 輸出類型：                                                   │
│  • 洞見卡片：標準輸出，記錄核心觀點                              │
│  • 衝突卡片：當發現矛盾觀點時                                    │
│  • 演變追蹤：當想法有變化時更新                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 彈性版本

### 忙碌日（15 分鐘版）

當時間有限時，只執行 Phase A + 最小輸出：

```
1. 掃描推播/Reader（5 分鐘）
2. 標記 1 篇「必讀」存起來
3. 快速寫一張卡片：只記標題 + 一句話想法（5 分鐘）
4. 完成

→ 不強求深讀，但保持「每日輸出」習慣
```

### 空檔穿插版（分段執行）

當沒有整塊時間時，分成小塊：

| 時段 | 動作 | 時間 |
|------|------|------|
| 早上通勤/等待 | Phase A：掃描、標記必讀 | 10 min |
| 午休/下午空檔 | Phase B：深讀 1 篇 | 15 min |
| 晚上/睡前 | Phase C：輸出卡片 | 10 min |

### 週末深度版（60-90 分鐘）

週末可以做更深入的處理：

```
1. 回顧本週的「稍後」標記，挑選值得讀的（10 min）
2. 深讀 3-5 篇（40 min）
3. 整理卡片、建立連結（20 min）
4. 週回顧：
   - 這週學到什麼？
   - 有哪些觀點改變了？
   - 資訊源需要調整嗎？
```

## 快速決策指南

### 「該讀還是略過？」

```
問自己這三個問題：

1. 這跟我當前的關注點有關嗎？
   → 無關 → 略過

2. 這可能改變我的想法或行動嗎？
   → 不會 → 略過

3. 這是我信任的來源嗎？
   → 不確定 → 放「稍後」，週末再處理
```

### 「該深讀還是快速掃過？」

| 情況 | 處理方式 |
|------|----------|
| 核心領域（醫學、AI） | 深讀 |
| 新觀點、挑戰既有認知 | 深讀 |
| 方法論、可應用的技巧 | 深讀 |
| 一般新聞、資訊更新 | 掃描標題即可 |
| 社群閒聊、八卦 | 略過或極快掃過 |

### 「該輸出什麼類型的卡片？」

```
讀完後問自己：

• 我學到了一個新觀點 → 洞見卡片
• 這跟我之前讀的矛盾 → 衝突卡片
• 我對這議題的看法變了 → 更新演變追蹤
• 只是有趣但沒新洞見 → 可以不輸出，或寫一句話筆記
```

## 狀態標記系統（Reader）

建議在 Reader 中使用以下 tag：

| Tag | 意義 | 何時用 |
|-----|------|--------|
| `#必讀` | 今日要深讀 | Phase A 標記 |
| `#稍後` | 有興趣但不急 | Phase A 標記 |
| `#已讀` | 已深讀完成 | Phase B 完成時 |
| `#已輸出` | 已寫成卡片 | Phase C 完成時 |
| `#略過` | 決定不讀 | Phase A 標記 |

### 狀態流

```
新文章 → #必讀 或 #稍後 或 #略過
             ↓
         #已讀（深讀完成）
             ↓
         #已輸出（有卡片產出）
```

## 每週回顧（週末 30 分鐘）

每週日花 30 分鐘做回顧：

### 1. 數據回顧（5 分鐘）
- 這週讀了幾篇？
- 輸出了幾張卡片？
- 時間有超支嗎？

### 2. 內容回顧（15 分鐘）
- 這週最重要的 3 個洞見是什麼？
- 有沒有觀點衝突需要記錄？
- 有沒有想法在演變？

### 3. 系統調整（10 分鐘）
- 哪些資訊源值得保留？
- 哪些可以取消訂閱？
- 流程需要調整嗎？

## 注意事項

### 不要做的事

- ❌ 強迫自己讀完所有未讀
- ❌ 在「掃描」階段陷入深讀
- ❌ 讀了但不輸出（這等於沒讀）
- ❌ 把「稍後」累積太多（每週清理）

### 要做的事

- ✅ 嚴格控制時間（設定計時器）
- ✅ 每天至少輸出一個東西
- ✅ 對「略過」毫不猶豫
- ✅ 週末處理累積的「稍後」
