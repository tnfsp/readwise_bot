# 每日資訊處理流程 (Workflow)

> 由 `/concept` 維護

---

## 設計原則

1. **彈性優先**：可以一次完成，也可以分段執行
2. **低門檻**：底線是每日 15 分鐘，其他是加分
3. **週輸出**：每週 3 張卡片，不強求每日
4. **允許略過**：忙碌日可以跳過，不累積壓力

---

## 分層設計

| 層級 | 時間 | 頻率 | 內容 |
|------|------|------|------|
| **最小可行** | 15 分鐘 | 每日 | 掃描 + 標記 |
| **標準版** | 30 分鐘 | 有空時 | 掃描 + 深讀 1 篇 |
| **完整版** | 50 分鐘 | 週末或特別有空 | 掃描 + 深讀 + 輸出 |

**核心承諾**：
- 每日底線：15 分鐘掃描
- 每週目標：3 張 Heptabase 卡片
- 深讀和完整輸出是「加分項」，不是必須

---

## 最小可行版（15 分鐘）- 每日底線

```
┌─────────────────────────────────────────────────────────────────┐
│  掃描 + 標記（15 分鐘）                                          │
│  ─────────────────────────────────────────────────────────────  │
│  目標：快速瀏覽，標記重點，維持習慣                              │
│                                                                 │
│  1. 打開 Reader（之後改用 Telegram 推播）                        │
│  2. 快速掃描標題                                                │
│  3. 標記：                                                      │
│     • 1-3 篇 #必讀（有空時深讀）                                 │
│     • 有興趣的 #稍後                                            │
│     • 其餘 #略過                                                │
│  4. 對特別有感的，在 Reader 寫一句話筆記                        │
│                                                                 │
│  ✅ 這樣就完成今日底線了                                         │
└─────────────────────────────────────────────────────────────────┘
```

**決策標準**：
- **必讀**：與當前關注相關 / 可能改變想法 / 可立即應用
- **稍後**：有趣但不急 / 需要更多時間
- **略過**：已知內容 / 與目標無關 / 信號弱

---

## 標準版（30 分鐘）- 有空時

```
掃描（10 min） → 深讀 1 篇（20 min）
```

深讀時：
- 在 Reader 中 highlight 重點
- 寫筆記：這篇的核心是什麼？我同意嗎？
- 讀完標記 #已讀

---

## 完整版（50 分鐘）- 週末或特別有空

```
掃描（10 min） → 深讀（25 min） → 輸出卡片（15 min）
```

輸出到 Heptabase：
- 洞見卡片：核心觀點 + 我的想法 + 信心度
- 衝突卡片：當發現矛盾觀點時
- 演變追蹤：長期議題的想法變化

---

## 週末深度版（60-90 分鐘）

```
1. 清理「稍後」：超過 2 週的，決定讀或刪（10 min）
2. 深讀 3-5 篇累積的 #必讀（40 min）
3. 整理本週卡片、建立連結（20 min）
4. 週回顧：
   - 這週學到什麼？
   - 有哪些觀點改變了？
   - 資訊源需要調整嗎？
```

---

## 快速決策指南

### 「該讀還是略過？」

```
問自己這三個問題：

1. 這跟我當前的關注點有關嗎？
   → 無關 → 略過

2. 這可能改變我的想法或行動嗎？
   → 不會 → 略過

3. 這是我信任的來源嗎？
   → 不確定 → 放「稍後」，週末再處理
```

### 「該深讀還是快速掃過？」

| 情況 | 處理方式 |
|------|----------|
| 核心領域（醫學、AI） | 深讀 |
| 新觀點、挑戰既有認知 | 深讀 |
| 方法論、可應用的技巧 | 深讀 |
| 一般新聞、資訊更新 | 掃描標題即可 |
| 社群閒聊、八卦 | 略過或極快掃過 |

### 「該輸出什麼類型的卡片？」

```
讀完後問自己：

• 我學到了一個新觀點 → 洞見卡片
• 這跟我之前讀的矛盾 → 衝突卡片
• 我對這議題的看法變了 → 更新演變追蹤
• 只是有趣但沒新洞見 → 可以不輸出，或寫一句話筆記
```

## 狀態標記系統（Reader）

建議在 Reader 中使用以下 tag：

| Tag | 意義 | 何時用 |
|-----|------|--------|
| `#必讀` | 今日要深讀 | Phase A 標記 |
| `#稍後` | 有興趣但不急 | Phase A 標記 |
| `#已讀` | 已深讀完成 | Phase B 完成時 |
| `#已輸出` | 已寫成卡片 | Phase C 完成時 |
| `#略過` | 決定不讀 | Phase A 標記 |

### 狀態流

```
新文章 → #必讀 或 #稍後 或 #略過
             ↓
         #已讀（深讀完成）
             ↓
         #已輸出（有卡片產出）
```

---

## 每週回顧（週末 30 分鐘）

每週日花 30 分鐘做回顧：

### 1. 數據回顧（5 分鐘）
- 這週讀了幾篇？
- 輸出了幾張卡片？
- 時間有超支嗎？

### 2. 內容回顧（15 分鐘）
- 這週最重要的 3 個洞見是什麼？
- 有沒有觀點衝突需要記錄？
- 有沒有想法在演變？

### 3. 系統調整（10 分鐘）
- 哪些資訊源值得保留？
- 哪些可以取消訂閱？
- 流程需要調整嗎？

---

## 注意事項

### 不要做的事

- ❌ 強迫自己讀完所有未讀
- ❌ 在「掃描」階段陷入深讀
- ❌ 把「稍後」累積太多（週末清理）
- ❌ 對自己太嚴格（彈性優先）

### 要做的事

- ✅ 嚴格控制時間（設定計時器）
- ✅ 每週至少 3 張卡片（不強求每日）
- ✅ 對「略過」毫不猶豫
- ✅ 週末處理累積的「稍後」
