# Readwise Reader 配置指南

> 由 `/pm` 維護

---

## 配置目標

1. **支援分層處理**：透過 tag 區分優先級
2. **追蹤狀態流**：待處理 → 已讀 → 已輸出
3. **減少決策疲勞**：用 Filter 快速進入工作狀態
4. **配合每日流程**：與 WORKFLOW.md 對應

---

## Tag 系統

### 處理狀態 Tag

| Tag | 意義 | 何時標記 | 顏色建議 |
|-----|------|----------|----------|
| `必讀` | 今日要深讀 | Phase A 掃描時 | 🔴 紅色 |
| `稍後` | 有興趣但不急 | Phase A 掃描時 | 🟡 黃色 |
| `已讀` | 深讀完成 | Phase B 完成時 | 🔵 藍色 |
| `已輸出` | 已寫成 Heptabase 卡片 | Phase C 完成時 | 🟢 綠色 |
| `略過` | 決定不讀 | Phase A 掃描時 | ⚫ 灰色 |

### 領域 Tag

| Tag | 用途 |
|-----|------|
| `醫學` | 醫學相關內容 |
| `AI` | AI/Claude Code 相關 |
| `國際` | 國際情勢 |
| `知識` | 知識管理/學習方法 |
| `生產力` | 工作流/效率 |
| `其他` | 不屬於以上分類 |

### 信號強度 Tag（可選）

| Tag | 用途 |
|-----|------|
| `高信號` | 核心來源，通常值得讀 |
| `中信號` | 偶有佳作 |
| `探索` | 嘗試性訂閱 |

## Filter 設定

### 建議建立的 Filter

#### 1. 「今日待讀」
```
條件：tag = 必讀
排序：最新優先
```
→ Phase B 時使用，只顯示今日要深讀的文章

#### 2. 「待處理」
```
條件：無 tag（或排除 已讀、已輸出、略過）
排序：最新優先
```
→ Phase A 掃描時使用，顯示尚未分類的新文章

#### 3. 「週末回顧」
```
條件：tag = 稍後
排序：最舊優先
```
→ 週末處理累積的「稍後」

#### 4. 「已完成」
```
條件：tag = 已輸出
排序：最新優先
```
→ 回顧自己的閱讀記錄

#### 5. 各領域 Filter
```
條件：tag = 醫學（或其他領域）
排序：最新優先
```
→ 當想專注某領域時使用

## 每日操作流程

### Phase A：掃描（在 Reader 中）

1. 進入「待處理」Filter
2. 快速瀏覽標題
3. 對每篇文章：
   - 點擊 → 加 tag `必讀` / `稍後` / `略過`
   - 同時加上領域 tag（如 `醫學`）
4. 完成後切換到「今日待讀」Filter

### Phase B：深讀（在 Reader 中）

1. 進入「今日待讀」Filter
2. 點開文章開始閱讀
3. 使用 highlight 標記重點
4. 使用筆記功能記錄想法
5. 讀完後加 tag `已讀`

### Phase C：輸出（跨 Reader 與 Heptabase）

1. 在 Reader 中回顧 highlight
2. 切換到 Heptabase，建立卡片
3. 回到 Reader，加 tag `已輸出`

## 與 Heptabase 的連動

### 建立卡片時

1. 複製 Reader 中的文章連結
2. 在 Heptabase 卡片中貼上作為來源
3. Readwise 的 highlight 可以自動同步到 Heptabase（如有設定）

### Readwise → Heptabase 同步設定

如果啟用 Readwise 官方整合：
- Highlight 會自動同步
- 可以在 Heptabase 中看到 Readwise 匯入的內容
- 適合作為卡片的「原料」

## 週末維護

每週日執行：

### 1. 清理「稍後」（15 分鐘）
- 進入「週末回顧」Filter
- 超過 2 週未處理的 → 改為 `略過`
- 仍有興趣的 → 立即閱讀或保留

### 2. 清理「略過」（5 分鐘）
- 定期封存（Archive）已略過的文章
- 保持收件匣乾淨

### 3. 檢視訂閱（10 分鐘）
- 哪些來源一直被略過？考慮取消訂閱
- 哪些來源持續產出好內容？標記為「高信號」
