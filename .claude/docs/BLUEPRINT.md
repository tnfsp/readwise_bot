# 個人訊息流強化系統 - 藍圖

> 系統全貌：資訊如何流入、處理、輸出

---

## 系統目標（重申）

```
不是為了讀更多、知道更多
而是為了判斷更好、行動更準確
```

**三個核心承諾**：
1. 每天 30-60 分鐘內完成
2. 讀過的東西要有輸出
3. 不確定的事情要標記出來

---

## 系統架構圖

### 核心決策：漸進式過渡（A → B）

**Phase 1：Reader 為主，推播為輔**
- Reader 保留完整內容（安全網）
- 推播做「今日精選提醒」
- 驗證 AI 篩選品質

**Phase 2+：推播為主，Reader 為深讀工具**
- 當信任建立後，減少打開 Reader 頻率
- 只有推播選中的才需要深讀
- FOMO 大幅降低

## 資訊流細節

### 1. 輸入層：如何收集

| 來源類型 | 數量 | 收集方式 | 進入點 |
|----------|------|----------|--------|
| RSS 訂閱 | ~70 | 自動同步 | Readwise Reader |
| Telegram 頻道 | ~20 | 手動或 RSShub | 直接閱讀 / Reader |
| PubMed 論文 | 15/次 | Pipedream 自動化 | LINE → 待遷移 |
| Newsletter | 若干 | Email 轉發 | Readwise Reader |

**問題**：來源太多（~90+），需要篩選機制

### 2. 處理層：如何篩選與閱讀

#### 每日推播（Telegram Bot）

```
06:00  系統自動執行
       │
       ├── 抓取過去 24h 新內容
       ├── AI 初篩：依領域分類、評估重要性
       ├── 產出摘要：每篇 2-3 句
       │
       ▼
07:00  推播到 Telegram
       │
       ├── 格式：領域 + 標題 + 摘要 + 連結
       ├── 數量：控制在 10-15 則
       ├── 互動：可標記「稍後讀」「略過」
       │
       ▼
       你決定今天要深讀什麼
```

#### 每日處理流程

| 階段 | 時間 | 動作 | 工具 |
|------|------|------|------|
| 掃描 | 10-15 min | 看推播、快速分類、決定焦點 | Telegram |
| 深讀 | 20-30 min | 閱讀 1-3 篇、畫重點、思考 | Readwise Reader |
| 輸出 | 10-15 min | 寫卡片、標記不確定、連結舊知識 | Heptabase |

### 3. 輸出層：如何整理與累積

#### Heptabase 卡片類型

**洞見卡片**（標準輸出）
```
標題：[主題]
來源：[連結]
日期：[YYYY-MM-DD]

核心觀點：
- ...

我的想法：
- ...

信心度：低 / 中 / 高
待驗證：
- [ ] ...
```

**衝突卡片**（當觀點矛盾時）
```
標題：[議題] - 觀點衝突

觀點 A：
- 來源：
- 主張：

觀點 B：
- 來源：
- 主張：

我目前的判斷：
- ...
- 為什麼：

待觀察：
- [ ] ...
```

**演變追蹤**（長期議題）
```
標題：[議題] - 我的理解演變

時間軸：
- [日期] 初次接觸，認為...
- [日期] 讀了 X，改變看法...
- [日期] 目前認為...

關鍵轉折點：
- ...
```

## 六大領域的處理策略

| 領域 | 頻率 | 深度 | 輸出形式 |
|------|------|------|----------|
| 知識/學習/筆記 | 每日 | 選擇性深讀 | 方法論卡片 |
| AI/Claude Code | 每日 | 重點追蹤 | 技術筆記、實作記錄 |
| 醫學 | 每日 | 論文級深讀 | 臨床應用筆記 |
| 國際情勢 | 每日 | 快速掃描 + 週末深讀 | 局勢理解卡片 |
| 生產力 | 低頻 | 有需要再讀 | 工作流改進 |
| 其他 | 探索性 | 隨機深讀 | 隨筆 |

## 實作階段

### Phase 1：基礎建立（方法論 + 手動執行）
- [ ] 確定資訊源架構（精簡 + 補強）
- [ ] 設計每日流程 SOP
- [ ] 設計 Heptabase 卡片模板
- [ ] 設計 Readwise Reader tag 系統
- [ ] 手動執行 2 週，驗證可行性

### Phase 2：推播自動化
- [ ] 建立 Telegram Bot
- [ ] 整合 RSS 篩選 + AI 摘要
- [ ] 遷移 PubMed 論文推播（LINE → Telegram）
- [ ] 設計互動機制（標記、回饋）

### Phase 3：進階優化
- [ ] AI 篩選調優（根據你的回饋學習）

## 成功指標

### 短期（1 個月）
- [ ] 每天 30-60 分鐘內完成資訊處理
- [ ] 每週產出 5+ 張 Heptabase 卡片
- [ ] FOMO 感受下降（主觀評估）

### 中期（3 個月）
- [ ] 資訊源穩定，不再頻繁調整
- [ ] 有 3+ 個議題建立了「演變追蹤」
- [ ] 能明確說出「這個我不確定，因為...」

### 長期（6 個月）
- [ ] 決策品質提升（可追溯到資訊來源）
- [ ] 建立 1+ 個領域的深度知識圖譜
- [ ] 系統運作穩定，維護成本低
